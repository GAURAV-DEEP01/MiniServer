MiniServer

About MiniServer

MiniServer is a lightweight C++ HTTP/1.1 server built from scratch. It's easily modifiable and features a simple routing API.

Features

  • Lightweight HTTP Server: Handles HTTP requests such as GET, POST, PUT, DELETE, and other methods using method-specific handlers.
  • Connection Keep-Alive: Supports HTTP/1.1 persistent connections.
  • Routing System: Define routes with lambda functions.
  • Customizable Responses: Set content type, status codes, and more.

Quick Start

Prerequisites

  • C++17 or later
  • A compatible compiler (G++)
  • Make build tool

Cloning the Repository

git clone // ill add later
cd Mini-Server

Building the Project

Using Make to build the project:

make build

This builds the project and outputs Webserver.exe inside an exe folder in your project directory.


If you dont have Make build tool use this
step 1: create a directory exe 

mkdir -p exe

step 2: build the project 

g++ -o exe/WebServer MiniServer/src/HttpServer.cpp MiniServer/src/Logger.cpp MiniServer/src/RequestHandler.cpp MiniServer/src/HttpRequest.cpp MiniServer/src/HttpResponse.cpp -lws2_32 server.cpp -IMiniServer/include

If you don't want to run the whole project, you can use it as a library instead. Additional documentation will be done soon...

Running the Server

./WebServer

Simple Example Usage

You can define routes and handle requests using the following API: 

// Inside the main create a HttpServer instance
HttpServer app;
                
                
// Define route method (There are routes for other HTTP methods too)
app.Get("/",[](Request &req, Response &res) -> int
{
res.setContentType("text/html");
res.send("Server is running!");
return 0;
});
                
// Listen on a port (This method invokation should be after all the 'route' definition)
app.listen(9090);

Now, check out localhost:9090/ on your browser.
